# Сервис доставки (вариант 6)

Учебная C4-модель по мотивам сервисов типа CDEK: посылки и доставки между пользователями.

---

## 1. Вариант и данные

| Сущность   | Назначение |
|-----------|------------|
| Пользователь | Отправители и получатели, учёт и поиск |
| Посылка   | Объект отправления, привязка к доставке |
| Доставка  | Маршрут от отправителя к получателю, связь с посылками |

**Требуемый API:** создание пользователя; поиск по логину и по маске имени/фамилии; создание посылки; получение посылок пользователя; создание доставки «от пользователя к пользователю»; получение информации о доставке по получателю и по отправителю.

---

## 2. Акторы и интеграции

- **Кто работает с системой:** отправитель, получатель, администратор.
- **С кем интегрируемся:** платёжная система, SMS-сервис, email-сервис.

---

## 3. Диаграммы в `workspace.dsl`

- **Контекст (C1):** система «Сервис доставки», три актора, три внешние системы и связи между ними.
- **Контейнеры (C2):** клиентское и админское веб-приложения, API Gateway, User / Parcel / Delivery / Notification / Payment сервисы, БД (PostgreSQL). Технологии и протоколы проставлены на контейнерах и связях.
- **Dynamic:** сценарий «Создание доставки от отправителя к получателю» — от запроса из клиента до сохранения в БД, оплаты и уведомления получателя.
